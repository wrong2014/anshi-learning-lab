P01 下游诊断系统六步工作记录
创建日期：2026-07-02
用途：防止断电或会话中断造成方向丢失；每完成一步都在本文件回写状态、决策、产物和遗留问题。

总目标
把已经生产的三层内部物料（知识图谱、三科核心思维、心理/认知/动机）变成 P01 能够安全调用的诊断能力。系统输出的是“当前最值得验证的学习卡点假设”，不是对孩子能力、人格或心理状态的定论。

六步总路线

[已完成：内部 v1] 1. 统一物料查询层（含向量化）
    把三层异构 JSON 规范成统一、可追溯的检索单元；建立精确过滤、全文/关键词检索、语义向量检索和知识图谱邻接扩展组成的混合检索。
    产出：版本化检索索引、查询 API、构建脚本、质量测试。

[未开始] 2. 诊断问题编译器
    先分析家长/孩子的真实场景，再从检索结果中编译“下一问”。不是把材料改写成题库；每个问题必须说明要区分哪些候选假设、需要什么证据、回答后如何更新判断。

[未开始] 3. 诊断智能体运行时
    将场景抽取、候选假设、证据账本、追问决策、规则校验、置信度更新和安全边界串成可恢复的会话流程。模型负责理解和表达，不能绕过规则内核。

[未开始] 4. 可解释结果与验证动作
    输出主假设、备选假设、支持证据、反证/缺失信息、置信度和“今晚 5 分钟验证动作”；不公开内部权重，不贴标签，不把相关性说成因果。

[未开始] 5. 接入 P01 产品
    将新内核接到现有 conversation API、生成式 UI、会话存储和 P02/P03 路由；保留无模型或检索失败时的安全降级路径。

[未开始] 6. 黄金案例与持续校准
    建立三科典型场景、近似混淆场景、反例、心理安全边界和回归集；用真实反馈校准检索、规则和追问，但不让模型凭少量案例自行改写诊断标准。

第 1 步关键判断：不是“只做向量库”

1. 可以并且应该向量化，但向量化只是统一查询层的一种召回方式。
2. 以下请求适合精确过滤/图查询：学科、年级、知识点 ID、前置/后置、关系类型、物料版本、公开使用权限。
3. 以下请求适合语义向量：家长的自然语言描述、孩子说法、同义表达、跨材料的相似可观察信号。
4. 中文短词、公式、符号和明确知识点名称容易被纯向量检索漏掉，所以必须保留全文/关键词检索。
5. 最终采用混合检索：metadata filter + lexical recall + dense vector recall + graph expansion；结果按来源层、场景匹配度和安全范围重排。
6. 每个检索单元必须保留 source_artifact、source_id、source_path、material_version、approval_scope，保证任何结论都能回到原文。

第 1 步切分原则

- 知识图谱：一个知识点概览一个单元；必要时按定义、学习要求、学科画像拆成子单元。图关系不盲目向量化，保留为结构化邻接边。
- 核心思维：定义/本质、各观察者的缺失表现、学段特征分别成单元，避免一个大段稀释“具体表现”。
- 心理认知：学科场景、家长信号、学生信号、低风险支持分别成单元；严重程度、转介条件和安全指南进入安全检索通道，不能被普通学习建议覆盖。
- 查询前先解析 subject、grade、actor、scene；无法确认时宁可放宽过滤并降低置信度，不武断猜年级或心理状态。

第 1 步当前已知风险

- 第一层正式知识图谱有知识点定义、学习要求、学科画像和关系，但没有系统化的“知识点级常见卡点/可观察信号”。因此第 1 步可以检索知识内容，却不能凭空声称已具备每个知识点的诊断信号。该缺口应在第 2 步前决定是从现有证据编译，还是新增受监督物料生产批次。
- 第二层、第三层目前是 Codex AI 内部使用条件批准，不是具名人类专家公开发布；索引必须保留使用范围并阻止被误当作公开心理诊断依据。
- 向量模型或索引版本变化会改变召回结果，必须记录 embedding_model、维度、文本摘要和构建时间，不能静默覆盖。

第 1 步验收标准

[x] 三层物料都能规范化，且数量与来源账本对得上。
[x] 每个检索结果可追溯到原始文件、对象 ID 和 JSON 路径。
[x] 支持 subject、grade、layer、unit_type、actor、approval_scope、safety_level 精确过滤。
[x] 支持中文自然语言语义查询，并同时保留关键词命中能力。
[x] 知识点结果可按前置/支撑/承接等关系扩展邻居。
[x] 心理安全/转介内容有独立 safety_level=referral 通道，可强制过滤检索。
[x] 代表性三科场景测试通过；错误召回和缺口已写回本文件。

第 1 步复盘与对第 2 步的约束

1. 分层检索效果可靠：数学表征、物理模型/论证、化学宏微符号、工作记忆、考试焦虑和安全转介均能命中对应来源。
2. 一次全库混搜不能直接当诊断结果。例如“物理换场景不会选公式”与考试焦虑场景都含“公式/不会”，若不先识别场景类型，心理材料可能被过早召回。
3. 因此第 2 步必须先做场景结构化，再分别查询知识、核心思维、心理安全三个通道，最后合并“候选假设”，不能拿全库 top-k 直接下结论。
4. 向量相似度只用于召回排序，不设通用绝对阈值；阈值必须通过后续真实案例校准。
5. 知识点级常见卡点/可观察信号仍是明确缺口。第 2 步可以先使用现有知识点 + 核心思维信号形成粗粒度追问，但不能声称已定位到某个知识点的具体错误机制。
