# 自然对话流与生成式 UI 设计 v01

整理日期：2026-06-26

## 目标

让家长感觉是在讲述一次真实学习事件，而不是在做测评题。系统在对话中穿插卡片、选择、滑条、短文本和可选孩子补充，后台完成结构化判断。

## 总体节奏

```text
轻边界 -> 选学科 -> 讲最近一次事件 -> 卡点选择 -> 学科追问
  -> 父母支持方式 -> 孩子可选补充 -> AI 关键追问
  -> 结果卡 -> 7 天动作 -> 推荐路径
```

## Step 0：轻边界

助手话术：

> 我们先不急着判断孩子“行不行”。我只帮你看最近一次理科学习卡住时，最可能卡在哪个环节。结果不是诊断，也不是提分承诺，而是一张家长可执行的支持方向卡。

UI：

- 按钮：开始定位
- 链接：查看边界说明

## Step 1：选择学科

助手话术：

> 今天先选一科。不要一次把所有问题都放进来，越具体越容易看清。

UI 单选：

- 数学
- 物理
- 化学
- 不确定，先从最近最困扰的一科开始

## Step 2：最近一次真实事件

助手话术：

> 想一个最近 2 周内的具体场景：一道题、一次作业、一次考试后复盘都可以。你不用写很完整，只要说“发生了什么、孩子怎么反应、你怎么帮”。

UI：

- 短文本输入
- 可选语音输入，V1 可先不做

后台抽取：

- subject
- scene_type
- parent_action
- child_response
- initial_factor_signals

## Step 3：卡住位置选择

助手话术：

> 如果只看这一次，孩子更像卡在哪一步？

UI 单选：

- 读不懂题 / 关键词不确定
- 概念或公式知道，但说不清
- 不知道怎么画图、列式或转化
- 不知道选哪个方法或公式
- 会做但步骤、计算、单位、检查总出错
- 看了答案懂，过两天又不会
- 一看到题就烦、急、想逃
- 说不清

后台映射：

- 读不懂题 -> F03
- 概念公式说不清 -> F02
- 画图列式转化 -> F04
- 选方法公式 -> F05
- 步骤计算检查 -> F06
- 看答案后复发 -> F07/F08
- 烦急逃 -> F09
- 说不清 -> F07 + 追问

## Step 4：学科特异追问

数学追问示例：

> 这道题如果换成例题同款，他会做吗？

物理追问示例：

> 他有没有尝试画过程图、受力图或电路图？卡在画图，还是卡在选公式？

化学追问示例：

> 他是现象看不懂、方程式写不出，还是知道方程式但不知道为什么这么反应？

UI：

- 单选 / 多选
- 题图上传预留

## Step 5：父母支持方式

助手话术：

> 你当时一般会怎么帮？这个问题不是判断你对错，是为了看家庭支持方式有没有和孩子卡点错位。

UI 多选：

- 直接讲完整解法
- 让孩子再读题
- 让孩子先说从哪一步不会
- 让 AI 讲给孩子听
- 让孩子问 AI 要答案
- 找老师或同学讲
- 加题量
- 先停下来复盘

后台映射：

- 直接讲完整解法 / AI 讲答案 / 加题量 -> F10/F08 风险
- 先说断点 / 复盘 -> 支持性证据

## Step 6：孩子可选补充

助手话术：

> 如果孩子愿意，可以只补充一句：这题你从哪一步开始不知道怎么往下走？不用解释对错。

UI 单选：

- 题目读完就不知道在问什么
- 知道题意，但不知道怎么画图或列式
- 知道要用公式，但不知道选哪个
- 公式选了，但代入和计算乱
- 答案看懂了，但下次还是不会
- 不想说 / 暂时跳过

## Step 7：AI 关键追问

追问只做 2-4 个，不无限聊天。

追问来源：

- 当前最高两个因子分数接近。
- 家长描述和选择冲突。
- 孩子补充与家长观察冲突。
- 出现 F09/F10 风险信号。

追问示例：

> 你前面说孩子“课堂听得懂”，但又说“题目一变完全不会启动”。更接近下面哪一种？

UI 单选：

- 例题同款能做，换情境不会
- 基础概念也说不清
- 题目条件读不准
- 会做但总错步骤

## Step 8：结果卡

结果结构：

1. 当前最可能主因。
2. 可能伴随因素。
3. 我们依据了哪些证据。
4. 还不能确定什么。
5. 父母容易帮错的一件事。
6. 未来 7 天先停止什么。
7. 未来 7 天先做什么。
8. 推荐路径。

## 结果话术原则

不要说：

> 你的孩子是建模能力差。

要说：

> 目前更像是“题目转化和建模迁移”先卡住了。证据是：例题同款能做，但换一个情境后，不知道从题干条件建立关系。未来 7 天先不要急着加题量，每天只选一道题，让孩子标出“从哪句话开始不知道怎么转”。
