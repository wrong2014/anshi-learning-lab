# P01 理科学习卡点定位智能体 MVP PRD v02

整理日期：2026-06-26

## 产品目标

通过与家长和孩子的自然对话，逐步收集学习过程证据，判断数学、物理、化学学习不稳定背后最可能的原因因子组合，并给父母一个可执行的下一步支持方向。

## 对外名称建议

对外不建议叫“诊断智能体”，可使用：

- 理科学习卡点定位助手
- 青少年数理化学习卡点定位工具
- 家庭学习支持定位助手

内部可以叫：

- P01 Diagnostic Agent
- 理科学习卡点定位引擎

## 核心判断

理科学不好的原因不是无限的，而是一个有限的原因因子集合。每个孩子当前的问题通常不是单一原因，而是主因 + 次因的组合。

本智能体要做的不是给孩子贴标签，而是回答：

1. 当前最可能卡在哪里？
2. 证据是什么？
3. 还缺什么信息？
4. 父母未来 7 天先停什么、先做什么？
5. 更适合进入 P02，还是需要 P03？

## 分科范围

首版支持三科：

| 学科 | 重点识别 |
|---|---|
| 数学 | 概念、符号、关系、抽象、迁移、步骤严谨性 |
| 物理 | 情境建模、过程图、变量关系、公式适用条件、单位和方向 |
| 化学 | 宏观现象、微观粒子、符号方程式、反应规律、实验解释 |

## 原因因子集合

首版使用 10 个内部因子：

| 编号 | 因子 | 简述 |
|---|---|---|
| F01 | 前置知识缺口 | 旧知识没接上，新内容自然不稳 |
| F02 | 概念理解不稳 | 会背定义公式，但没有真正理解 |
| F03 | 学科语言与符号理解困难 | 题干、术语、符号、图像读不准 |
| F04 | 表征转换困难 | 文字、图、式子、过程之间转不过来 |
| F05 | 建模与迁移困难 | 例题会，换情境不会建立关系 |
| F06 | 程序执行不稳定 | 审题、步骤、计算、单位、书写、检查失控 |
| F07 | 元认知与复盘薄弱 | 不知道自己哪里不会，也不会纠错 |
| F08 | 学习策略低效 | 只刷题、看答案、背公式，没有闭环 |
| F09 | 情绪动机与自我效能受损 | 怕错、逃避、焦虑、觉得自己不行 |
| F10 | 家庭支持与 AI 使用失位 | 父母越帮越乱，AI 变答案机 |

## 自然对话原则

用户不喜欢做题，所以前台不要呈现成测评卷。应呈现为“聊最近一次真实学习事件”。

前台体验：

```text
家长讲一个最近场景
  -> 智能体用卡片帮他归纳
  -> 穿插 1-2 个选择题降低输入成本
  -> 邀请孩子可选补充一句
  -> 智能体追问关键分歧点
  -> 输出学习卡点定位结果
```

后台逻辑：

```text
学科分流
  -> 证据抽取
  -> 因子评分
  -> 不确定点追问
  -> 风险检查
  -> 结构化结果
  -> 家长可读解释
```

## 交互形式

生成式 UI 必须参与，不做纯文本聊天。

首版需要的 UI block：

| 类型 | 用途 |
|---|---|
| 单选卡片 | 选择学科、最近卡住环节、AI 使用方式 |
| 多选 chips | 选择常见表现、父母常见帮助方式 |
| 程度滑条 | 频率、冲突强度、孩子回避程度 |
| 短文本输入 | 让家长描述最近一次具体事件 |
| 孩子补充卡 | 可选，问孩子“从哪一步开始卡住” |
| 结果卡 | 主因、次因、证据、7 天动作 |

## 不做目标

- 不输出“孩子属于某某类型”。
- 不输出心理诊断。
- 不承诺提分。
- 不直接给完整解题过程。
- 不让家长看到完整权重和内部因子映射。
- 不做无限聊天。

## 首版流程

1. 边界说明：这是学习卡点定位，不是诊断或提分承诺。
2. 选择学科：数学 / 物理 / 化学。
3. 家长描述最近一次真实学习事件。
4. UI 选择：孩子卡在读题、理解、列式、计算、复盘、情绪、AI 使用等哪个环节。
5. 学科特异追问：数学、物理、化学分别问不同问题。
6. 父母支持方式追问：父母通常如何帮。
7. 孩子可选补充：只问学习过程，不问人格和隐私。
8. 规则初判：形成主因候选和次因候选。
9. AI 追问：只追问 2-4 个最能区分因子的点。
10. 输出结果页：当前最可能原因组合 + 证据 + 7 天动作 + 推荐路径。

## 推荐路径规则

| 情况 | 推荐 |
|---|---|
| 家庭支持与 AI 使用失位为主，材料复杂度不高 | P02 |
| 存在多因子叠加，且外部资源投入后仍无效 | P03 |
| 情绪冲突严重、孩子强烈拒绝沟通 | 人工初筛，不直接承接 |
| 只要提分、答案、题库 | 不适配，温和说明边界 |
