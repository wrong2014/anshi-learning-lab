# P01 Diagnostic Agent 工作指南

本文档是代码仓库中面向 AI 智能体（Agent）的全局入口地图。当你作为智能体介入这个项目时，请优先通过本目录找到你需要遵循的上下文约束。

## 目录结构映射

- `docs/core-beliefs.md`: P01 教育产品的“铁律”和底线原则。任何针对大模型 Prompt 或底层规则的修改，都绝对不能违反这里的理念。
- `docs/exec-plans/`: 历史执行计划和待处理技术债。如果你不知道当前该做什么，或者发现上下文断裂，可以查阅此目录获取历史决策信息。

## 架构原则 (Taste Invariants)

1. **不可绕过规则引擎**：大模型的输出必须与 `factor_rules.py` 进行 Grounding 交叉验证。如果不一致，必须降级置信度或内部发起重试。
2. **结构化强制交互**：只要有可能，就用 UI 组件（如 `single_choice`）让用户点击，而不是放任自由文本输入。
3. **零手工代码理念**：任何架构优化、Bug 修复和测试用例，都应由智能体生成。不要建议用户手动复制粘贴代码。

请在编写任何业务代码前，确认你已阅读相关的记录系统（System of Record）文件。
