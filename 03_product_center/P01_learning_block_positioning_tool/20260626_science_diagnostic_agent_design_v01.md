# 理科学习卡点定位智能体设计方案 v01

整理日期：2026-06-26

## 一句话定义

理科学习卡点定位智能体通过家长和孩子的自然对话，收集可观察学习过程证据，识别数学、物理、化学学习不稳定背后最可能的原因因子组合，并生成父母可执行的 7 天支持动作。

## 设计原则

1. 原因集合有限，体验不能像做题。
2. 核心逻辑藏在内部规则、因子矩阵和追问策略中。
3. 前台是聊天 + 生成式 UI，后台是结构化判断。
4. 模型不直接决定最终分类，模型负责抽取、追问、解释和润色。
5. 结果是“当前最可能支持方向”，不是心理或能力诊断。

## 智能体边界

它可以：

- 识别主因和次因。
- 找出还缺的信息。
- 给父母下一步支持动作。
- 推荐进入 P02 或 P03。

它不可以：

- 给孩子贴人格标签。
- 输出医学或心理诊断。
- 替孩子解题。
- 承诺提分。
- 公开展示内部评分权重。

## 内部模块

| 模块 | 作用 | 是否暴露给用户 |
|---|---|---|
| Subject Router | 判断数学、物理、化学路径 | 只暴露学科选择 |
| Evidence Collector | 从对话中提取证据 | 不暴露 |
| Factor Scorer | 对 10 个原因因子评分 | 不暴露 |
| Question Planner | 决定下一步追问什么 | 不暴露 |
| Risk Guard | 判断是否需要人工或不适合承接 | 不暴露 |
| Result Composer | 把结构化结论写成家长听得懂的话 | 暴露 |
| Conversion Router | 推荐 P02 / P03 / 暂不适配 | 部分暴露 |

## 前台体验

用户感觉自己在和一个懂家庭学习场景的人聊天：

```text
家长：孩子物理课堂能听懂，一做题就不会。
助手：我先不急着判断。我们只看最近一次：他卡在读题、画图、选公式、代入计算，还是检查？
UI：读题 / 画图 / 选公式 / 计算 / 检查 / 说不清
```

后台同时发生：

```text
subject = physics
signals = ["听懂但做题不会"]
candidate_factors = [F04, F05, F07]
next_question = 区分“表征转换困难”与“建模迁移困难”
```

## 为什么每个因子需要不同诊断方法

不同因子对应不同证据类型：

| 因子类型 | 需要的证据 |
|---|---|
| 前置知识缺口 | 是否旧知识调用不出来 |
| 概念理解不稳 | 是否能用自己的话解释 |
| 表征转换困难 | 是否能把文字、图像、公式互相翻译 |
| 建模迁移困难 | 是否能识别变量关系和适用条件 |
| 执行不稳定 | 是否步骤、单位、计算、检查反复失控 |
| 元认知薄弱 | 是否知道自己从哪一步不会 |
| 策略低效 | 是否有复盘和复测闭环 |
| 情绪动机受损 | 是否出现回避、怕错、自我否定 |
| 家庭支持失位 | 父母和 AI 是否在替代孩子思考 |

因此不能用一张普通问卷覆盖全部。必须根据对话中出现的信号，动态选择追问。

## 结论结构

输出必须固定为结构化结果：

```json
{
  "subject": "physics",
  "primary_factor": "F05",
  "secondary_factors": ["F04", "F07"],
  "confidence": "medium",
  "evidence": [],
  "missing_information": [],
  "parent_common_mistake": "",
  "next_7_days_stop": "",
  "next_7_days_start": "",
  "recommended_path": "P02",
  "human_review_needed": false
}
```

## V1 完成标准

- 能支持数学、物理、化学三科分流。
- 能用 10 个内部因子评分。
- 能根据不同因子给不同追问。
- 能生成可解释结果页。
- 能在无 LLM key 的情况下跑通本地规则 demo。
- LLM key 接入点留空，不阻塞产品设计。
