## 一、项目概述

开发一个 AI 驱动的学习卡点定位诊断工具（P01），通过自然对话方式帮助家长发现孩子学习中的底层卡点（知识断层、认知瓶颈、策略失效等），输出个性化诊断报告和可操作的改善建议。

**技术栈**: Python (FastAPI/agent engine) + React/TypeScript (frontend) + LLM

## 二、问题发现与修改清单（按时间顺序）

### 🔴 问题 1: 诊断对话准确性不足 (第1~3轮迭代)

**发现的问题**:
- 对话流程过于死板，12 因子诊断矩阵 + 固定 6 轮追问导致用户疲惫
- LLM 输出与规则引擎信号冲突，出现矛盾诊断
- 自由文本输入的路由不合理：所有无法匹配的输入都归入 `F07`（学习策略），导致诊断偏倚
- 多选 UI 组件误触发：仅凭单词匹配（如 "AI" 触发某个选项），假阳性高

**做的修改**:
- **LLM/规则双路径融合 + 交叉校验** (`d00a4a6`)
  - `_record_rule_signals()` 提升到 `is_ready()` 之前，规则模式外也收集信号
  - 新增 `_ground_with_rules()`：LLM 出结论后用 `accumulate_scores` 做兜底校验
  - 修复 `combined_text` 将 UI 标签文本误喂给 `infer_option_ids_from_text` 导致假 option_id 注入的 bug
- **自由文本信号路由 + 自适应追问** (`d00a4a6`)
  - `engine.py score_session`：自由文本用 `infer_option_ids_from_text` 提取信号，不再全部归到 F07
  - `question_bank.py`：新增 F01/F02/F03/F06/F11/F12 共 6 个自适应追问
  - `factor_rules.py`：新增 18 个 option_id 权重 + 公共标签
  - `llm_providers.py`：同步 `ALLOWED_OPTION_IDS`
- **先验衰减 + 多选正则收紧** (`d00a4a6`)
  - `accumulate_scores`：学科先验随证据数量线性衰减，4+ 选项时归零
  - `_normalize_ui_block`：多选判断从单关键词改为多模式组合，避免单词误触发

### 🔴 问题 2: 追问选项不区分年级（小学/初中/高中混合） (第4~5轮)

**发现的问题**:
- 所有年级共用同一套探针选项，导致小学低年级遇到"小数分数转换"问题，初中生看到"进退位"问题
- "初一"被错误提取为"一年级"，年级映射完全失效
- 钩子文案（引流话术）对高中生说"20以内进退位"，完全不合时宜

**做的修改**:
- **年级感知系统** (`63c560c`)
  - `ConversationSession` 新增 `fallback_grade` 字段
  - `_extract_grade()`：从文本提取年级（一年级~高三）
  - `_grade_bucket()`：年级分桶 `lower/middle/upper/junior/senior`
  - `_GRADE_FILTER_OPTIONS`：低年级自动过滤小数点/多步等高年级选项
  - `_fallback_subject_probe` 按年级过滤不合适的探针选项
  - `PERSONALIZED_ADVICE` 使用 `{grade_opener}` 占位符动态替换
  - `HOOKS` 使用 `{grade_hook}` 占位符，按年级桶输出对应追问
- **初中/高中年级提取修复** (`335a0ae`)
  - `_extract_grade`：先匹配初中/高中表达式，后匹配小学——"初中一年级"→7, "高中二年级"→11
  - `_GRADE_FILTER_OPTIONS`：新增 `junior/senior` 过滤规则——初中过滤进退位/抄错数字等小学选项，高中过滤更多
  - 补全初中/高中钩子文案

### 🔴 问题 3: 追问路径死板，结果输出学术腔重 (第6轮)

**发现的问题**:
- 追问方向固定，不根据用户已确认的卡点动态调整
- 学科探针选项太少，只能覆盖数学，无法扩展到物理/化学
- 诊断结果像论文摘要，家长看不懂、记不住、不会用
- 缺少引流机制——诊断完就结束了，用户不知道下一步做什么

**做的修改**:
- **动态学科探针** (`0182ca6`)
  - `_fallback_subject_probe` 根据 `stuck_step` 生成针对性追问
  - 新增 60 个学科探针选项（数学/物理/化学 × 8 种卡点）
  - `_DYNAMIC_PROBES` 按 `stuck_step` 匹配追问方向
- **口语化结果 + 引流钩子** (`0182ca6`)
  - `_PERSONALIZED_ADVICE`：每种卡点有专用 `why_happens` + `try_this`
  - `_CAUSAL_LINKS`：口语化因果推理（如"投篮姿势不对投100个还是错的"）
  - `_HOOKS`：按学科/通用埋引流钩子
  - 开场白/追问文案改为温暖口语风格
  - 追问环节保持学术专业，结果输出白话+钩子

### 🔴 问题 4: 诊断框架过度复杂，核心卡点被稀释 (第7轮)

**发现的问题**:
- 12 因子诊断矩阵平均分配权重，每个因子分值都很低，核心卡点被噪声淹没
- 6 轮固定追问让家长不耐烦，完成率低
- "家长辅导方式"作为独立主因不合理——它只是放大器，不是根因
- 诊断报告的结论含糊（"可能是这个也可能是那个"），缺乏可操作性
- 学科覆盖面宽但精度差（泛而不精）

**做的修改**:
- **V2 初筛智能体架构重构** (`d22979f`)
  - **12因子 → 5主卡点(A-E) + 3放大器(F-H)**：聚焦核心卡点
    - A: 概念理解断裂
    - B: 思维技能缺失
    - C: 认知加工瓶颈
    - D: 学习策略低效
    - E: 情绪-动机阻滞
    - F: 家庭支持失位（降为放大器）
    - G: 教学适配不当（降为放大器）
    - H: 生理节律失调（降为放大器）
  - **固定6轮追问 → V2五步状态机**：复述→缩小→类别追问→放大器→报告
  - 家长辅导方式从主因降为放大器（权重系数 0.5）
  - 新增 `StuckCategory`/`Amplifier` 枚举 (`models.py`)
  - 新增 `accumulate_category_scores` 类别聚合评分 (`factor_rules.py`)
  - 新增 `verification_actions.py`：5分类×2学科 5分钟验证动作
  - 重写 `prompts.py`：V2 完整 System Prompt
  - 精简 `question_bank.py`：`category_probe` + `amplifier_probe`
  - 重写 `conversation_agent.py`：V2 状态机 + 5段报告模板
  - 报告强制包含：还不能确定 / 今晚验证 / 试卷精诊钩子
  - 删除旧标签：不再输出"学习策略低效""家庭支持失位"
  - 学科缩减为数学+物理

### 🔴 问题 5: 报告文案缺乏冲击力，用户不易行动 (第8轮)

**发现的问题**:
- 诊断报告缺少"犀利句"——用户看完不痛不痒，不转发不收藏
- 缺少共情环节——家长可能已经在自责，报告直接砸结论会引发防御
- 缺少"证据转化"——家长口述 → 试卷验证的路径不清晰

**做的修改**:
- **V2 报告文案补齐** (`eb9d631`)
  - 报告开头："别再无效刷题，这是底层逻辑架构没搭稳"（盖楼比喻）
  - 验证后共情："注意你大概率会发现孩子说不出第一处跑偏点"（缓解焦虑）
  - 试卷转化："看病不能光听描述必须看化验单"（X光扫描比喻）
  - 追加模糊追问："印象最深的一次"
  - Server 标题改为 `MathScreeningAgent/2.0`
  - 保留数学+物理双学科

### 🔴 问题 7: 答非所问——用户说不确定时仍被强制追问 (第10轮)

**发现的问题**:
- 用户选择"都不像"选项时，系统仍然强制追问，体验极差——用户说"都不是"，系统却继续追问，像机器人
- 复述文案（parent paraphrase）被丢弃，用户看不到系统是否理解正确
- 文本信号匹配模式不够，部分真实卡点无法识别
- 窄化文案缺少免责声明——如果列表都不匹配用户不知怎么办

**做的修改**:
- **答非所问修复** (`2a4d98b`)
  - `_paraphrase_parent` 复述文案之前被丢弃，现已拼入响应文本（让用户确认理解是否正确）
  - `TEXT_SIGNAL_PATTERNS` 从 12 条扩至 18 条，覆盖：概念不懂/应用题/读题/考试/情绪等场景
  - `narrow_unsure`：选"都不像"时不再强制追问，改为自由描述（"直接告诉我你的感觉"）
  - 窄化文案加免责："如果都不像，请选'不确定'直接说"
