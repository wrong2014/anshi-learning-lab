# AI 生产系统总说明 v01

整理日期：2026-06-26

## AI 的位置

AI 是后台生产引擎和父母支持工具，不是产品主角，也不是孩子的答案机。

## P01 技术路线

首版采用：

```text
规则评分 + 结构化 AI 追问 + 固定结果模板 + 少量个性化解释
```

暂不采用多 Agent 表演式协作。

## 工作流

```text
父母输入
  -> 基础准入与风险筛查
  -> 规则初步定位
  -> AI 追问 2-4 个关键问题
  -> 结构化结果
  -> 结果页和转化路径
```

## 结构化输出字段

```json
{
  "primary_support_direction": "题目转化与迁移卡住",
  "confidence": "medium",
  "evidence": [],
  "parent_common_mistake": "",
  "next_7_days_action": "",
  "recommended_path": "P02",
  "risk_flag": false,
  "human_review_needed": false
}
```

## 内部 AI 角色

| 角色 | 可做 | 禁止 |
|---|---|---|
| 材料整理员 | 整理错题、卷子、家长描述 | 直接给最终结论 |
| 证据归纳员 | 发现重复现象和矛盾 | 给孩子贴人格标签 |
| 方案草拟员 | 按框架起草行动方案 | 替人决定优先级 |
| 工具造物师 | 生成轻量工具原型 | 擅自扩大数据范围 |
| 风险检查员 | 检查越界和隐私风险 | 替代人工最终判断 |

## 评测要求

- 每次分类必须能追溯到输入证据。
- 结果页不得制造诊断感。
- 风险标记必须优先保守。
- 不确定时推荐人工审核。
- 所有测试样本必须脱敏。
