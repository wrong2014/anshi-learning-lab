## 生产中发现并修正的问题

1. 断电前生成请求脚本发生编码破坏，已重写为 UTF-8 并通过语法检查。
2. 专用智能体提示词存在历史乱码，导致模型实际接收的职责与量表不可读；已重写 Planner、Executor、Supervisor 提示。
3. 首次数学候选把关系目标写成中文名称，旧验证器未判错；现已要求关系只引用当前物料中真实存在的稳定 ID，并检查自环和支持环。
4. “学段进阶”被规划成过大的重复批次，导致维度膨胀和证据稀释；数学任务已收敛为三个核心维度，每个维度内部覆盖完整进阶。
5. 增加三类观察者必须齐全、学段不得有空档或重叠、批次模块必须与维度一一同名等硬校验。
6. 物理专用 Schema 在 Flash 模型上连续出现不可解析输出；保留双独立候选原则，改为两路不同采样参数的 Pro 执行器。
7. 三科原始局部 ID 存在同名碰撞；工作包在合并时统一加 `math.`、`physics.`、`chemistry.` 命名空间，14 个 ID 全局唯一。
