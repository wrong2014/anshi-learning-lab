# 青少年数理化学习问题资料生产系统：设计回溯与修正基线 v1.1

状态：执行中

建立日期：2026-06-29

依据：最初《完整设计方案 v1.0》、教育部义务教育课程标准（2022 年版）及当前代码审计

## 1. 文档目的

本文件同时承担三项职责：

1. 记录最初设计、后续决策与当前实现之间的差异；
2. 规定后续代码修正的唯一目标架构，防止再次发生概念漂移；
3. 作为验收清单和变更记录，未满足本文件验收条件的能力不得宣称“完成”。

## 2. 不变的产品目标

系统生产七类资料：

1. 数学知识图谱与学习卡点；
2. 物理知识图谱与学习卡点；
3. 化学知识图谱与学习卡点；
4. 数学核心思维；
5. 物理核心思维；
6. 化学核心思维；
7. 青少年学习心理与认知发展。

每一类资料都必须遵守 Planner、Executor、Supervisor 三层质量架构。模型端点可替换，密钥只能从环境变量读取。确定性错误必须先由代码拦截，不能交给 Supervisor 用评分掩盖。

## 3. 本次回溯发现的偏离

### 3.1 概念偏离

最初设计中的“最难的三个智能体”指数学、物理、化学三个知识图谱资料生产智能体。后续实现把“大纲、知识点、关系”称为三个智能体，导致领域智能体和生产阶段混为一谈。

修正决定：

- 数学、物理、化学仍是三个领域生产智能体；
- 大纲、知识点、关系是知识图谱 Executor 内部的三个阶段；
- 三阶段不能替代 Planner、并行 Executor、Supervisor。

### 3.2 流水线分叉

审计时，通用 `ProductionFactory` 已具备多模型候选、Supervisor、90/70 决策、有限重跑、SQLite 留痕和发布；当时的课标 `CurriculumPipeline` 只有单模型串行阶段。该偏离现已按本文件修正。

修正决定：保留两边有效能力，将课标流水线收拢为知识图谱领域智能体的专用生产流程，并复用统一的监督与持久化原则。

### 3.3 数据契约过于通用

通用 `KnowledgeNode` 无法完整表达原设计中的学科特性。

修正决定：知识点增加结构化学科画像：

- 数学：定义、表征、程序、应用等知识类型；
- 物理：概念定义、公式表达、概念与公式关联、跨模块依赖；
- 化学：记忆/推理/混合类型、宏观—微观—符号表征转换；
- 所有学科：前置、后置、迁移、承接关系由图边表达，不在自由文本中重复维护。

### 3.4 范围变化

最初设计包含初高中；当前明确任务是小学到初中。

修正决定：v1.1 的正式范围为中国义务教育 1-9 年级。高中不是删除，而是后续扩展阶段；任何文档不得继续把当前产物描述为“初高中完整覆盖”。

## 4. 修正后的目标架构

```text
领域智能体（数学 / 物理 / 化学）
  -> Planner
       官方课标来源与 OCR 证据
       不可删改课标种子
       完整学段任务树
  -> Executor Stage A：大纲细化
       多模型独立候选
  -> Supervisor A：大纲候选审查
  -> Executor Stage B：逐叶知识点提取
       每个叶子至少两个模型独立候选
  -> 硬校验 B：范围、页码、摘录、覆盖、学科字段
  -> Supervisor B：择优、局部重跑或整体重跑
  -> Executor Stage C：关系建网
       多模型独立候选
  -> 硬校验 C：悬空边、自环、重复边、前置环、科学承接
  -> Supervisor C：择优和重跑
  -> 最终合并与全局覆盖审查
  -> SQLite 运行留痕 + 可恢复检查点
  -> 不可变 Knowledge Network 发布包
```

Supervisor 是语义质量判断者；覆盖数量、引用真实性和图结构有效性由代码决定。

## 5. 七类资料的实现标准

### 5.1 数学、物理、化学知识图谱

必须满足：

- 课程边界来自官方课标，不允许 `model_distillation` 自行确定范围；
- 每个学段任务均有知识点覆盖账本；
- 每个生产阶段至少两个独立 Executor 候选；
- 每个候选先过硬校验，再进入 Supervisor；
- 物理和化学必须使用专用字段；
- 小学科学必须分别连接到初中物理和化学；
- 完成后生成可供学习卡点与诊断追问编译器消费的统一产物。

### 5.2 三类核心思维资料

不能仅使用知识点 Schema 代替。至少需要：学术定义、通俗本质、学段特征、缺失表现、依赖/支撑思维、学科边界、可观察信号和来源。

### 5.3 心理与认知资料

至少需要：真实理论来源、数理化场景、家长/学生可观察信号、关联机制、严重程度、AI 支持边界、专业转介条件。必须来源审计并经过人工审批。

## 6. 保留的增强能力

以下能力虽超出 v1.0，但属于正确增强，不回退：

- 教育部官方来源目录、SHA-256 与逐页 OCR；
- 确定性课标种子和模型不可删改约束；
- Pydantic 严格 Schema；
- 覆盖账本和图结构硬校验；
- 禁止“粗心”“不努力”等主观标签；
- SQLite 审计、不可变发布包和摘要校验；
- 心理资料人工审批闸门；
- 分阶段检查点和中断续跑。

## 8. v1.1 验收条件

本阶段只有同时满足以下条件才可称为完成：

1. 课标知识图谱三阶段均有至少两个独立候选和 Supervisor；
2. Supervisor 的 pass/partial/full 决策与 90/70 阈值由 Schema 和代码共同约束；
3. 每个阶段支持有限重跑和中断续跑；
4. 物理、化学专用字段存在并通过硬校验；
5. 最终网络可以持久化、验证和发布；
6. 单元测试、流程测试和失败路径测试全部通过；
7. 文档明确当前只覆盖 1-9 年级。
